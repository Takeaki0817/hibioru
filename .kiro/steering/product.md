# プロダクト概要

**サービス名**: ヒビオル (hibioru)
**コンセプト**: 「日々を織る」 - 1日の中での心の動きを記録する思考記録アプリ

Twitter的な投稿感覚で、日々の瞬間をログとして残す。

**開発の背景**:
開発者自身がADHDの気質により、過去20年間、日記や日報の継続に失敗してきた。
「書かなきゃ」と思っても、いざ開くと書けない。気づいたら数日空いて、やる気がなくなる。
この経験から、「続けやすさ」に特化したアプリを作った。

**目的**:
- 考えていることを言語化して記録する習慣を作る
- 誰にも見せない、雑に吐き出せる場所を作る
- 記録をAIも使って振り返りやすくする
- **充実感を正確に認識する** - 記録がないと「何もしていない」と錯覚しがち

**参考**: [noteの記事](https://note.com/takeaki0817/n/nde4b568e21b3)

## 5つの理想条件

ADHD当事者が「続けられる」ツールの条件：

1. **どこでも** - スマホ・PC・タブレット、どのデバイスからでも
2. **気楽に** - 絵文字1つ、一言だけでOK
3. **待たずに** - 2タップで記録完了、ロードを待たない
4. **誰にも見られず** - 完全プライベート、共有は任意
5. **ツールトラブルなく** - PWAでインストール不要、データはクラウド同期

## 筆記開示

心理学では「筆記開示（エクスプレッシブ・ライティング）」と呼ばれる手法がある。
思考を書き出すことで、以下の効果が期待できる：

- 頭の中のモヤモヤが整理される
- 感情が外在化され、客観視できる
- ストレスや不安の軽減
- 夜の考え事が減り、睡眠の質が向上

ヒビオルは、この筆記開示を「続けられる形」で実現するためのアプリ。

## コア機能

### 1. ストリーク（継続記録）システム
Duolingo式の継続モチベーション設計
- 毎日0:00リセット
- 1件以上の記録で継続カウント
- 「失いたくない資産」としての心理効果

### 2. ほつれ（スキップ機能）
継続を守るためのセーフティネット
- 週2回付与、繰り越しなし
- **自動消費**: 記録がない日に自動で使用
- 完全な失敗を防ぐ「途切れても大したことない」感

### 3. 最小単位入力
「やらない理由」を潰す設計
- 絵文字1つでもOK
- 2タップで記録完了
- 画像添付可能（1投稿1枚）

### 4. プッシュ通知
ADHD当事者の「忘れる」「自分から動けない」への対策
- 定時リマインド + 追いリマインド
- 記録するまで最大3回通知

## こんな悩みを持つ人へ

- 気がついたら1日、1週間、1年が過ぎて、何をしていたかあまり覚えていない
- 日記や日報を続けようとしたけど、続いたことがない
- 人に話せないことや、読んでもらうための文章にするのが面倒くさい
- 頭の中のモヤモヤを書き出してスッキリしたい
- 愚痴や怒り、悲しみを吐き出す場所がほしい
- 考え事で集中できない、眠れないときに、頭を空っぽにしたい
- 記録したデータをAIを使って振り返りたい

## 価値提案

### Duolingoから学んだ設計思想

1. **損失回避** - ストリークが「失いたくない資産」になる
2. **最小単位が極端に小さい** - 「やらない理由」が潰される
3. **セーフティネット** - ほつれで完全な失敗を防ぐ
4. **可視化** - 記録が目に見える形で残る

### 差別化ポイント

- 「瞬間記録 x 継続支援」の組み合わせ
- Duolingo式ゲーミフィケーション
- ADHD当事者が設計（当事者視点）

| 用語 | 意味 |
|------|------|
| **ヒビオル** | サービス名。アニメ映画『さよならの朝に約束の花をかざろう』に登場する布。縦糸が「月日」、横糸が「人のなりわい」を表し、日々の出来事や人生そのものを意味する |

## 設計原則

### やらない設計

ADHDの報酬系を余分に刺激しないため、意図的に排除した機能：

| 機能 | 排除理由 |
|------|---------|
| **DM機能** | 返信義務感がストレスになる。記録に集中できなくなる |
| **通知数バッジ** | 数字が気になって他のことに集中できない |
| **フォロワー数の外部表示** | 数字の比較がストレスになる |
| **無限スクロールフィード** | ドーパミンループに陥りやすい |
| **いいね数の表示** | 承認欲求の刺激を避ける |
| **おすすめ・トレンド** | 過剰な情報摂取を避ける |

報酬は「継続」に対してのみ与える設計。

### 守るべきこと
- 報酬は「継続」に対してのみ
- 無料で十分使えることを維持（課金圧を強くしない）
- ADHD当事者にストレスを与えない設計を優先
