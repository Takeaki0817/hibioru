## このドキュメントの目的

## プロジェクトの発端

### 最初の問い

「Twitterの様な投稿感覚で日記ならぬ瞬間の記録をログとして残せるアプリってある？スマホとPCどこからでもアクセスできて、かつ、AIにその日の記録を渡すことができれば理想」

### なぜ自作するのか

開発者自身がADHDの気質により、**過去20年間**、日記や日報の継続に失敗してきた。
既存ツール（Obsidian、Notion、Day One等）を検討したが、ADHD当事者にとっての「継続」という課題に特化したものがなかった。

### ADHD当事者が抱える5つの悩み

1. **絶望的に忘れっぽい** - 気がついたら1日、1週間、1年が過ぎている
2. **飽き性で継続が困難** - 日記や日報を続けようとしても、続いたことがない
3. **気になったことを横目に置けない** - 集中中に別の思考が割り込む
4. **夜に考え事で寝れない** - 頭の中がうるさくて眠れない
5. **その結果、精神的に落ち込む** - 「何もしていない」と錯覚しがち

---

## 筆記開示（エクスプレッシブ・ライティング）

心理学では「筆記開示」と呼ばれる手法があり、思考を書き出すことで以下の効果が期待できる：

- 頭の中のモヤモヤが整理される
- 感情が外在化され、客観視できる
- ストレスや不安の軽減
- 夜の考え事が減り、睡眠の質が向上

**問題は「続けることが難しい」こと**。だから、続けやすさに特化したアプリを作った。

---

## 5つの理想条件

ADHD当事者が「続けられる」ツールの条件として導き出したもの：

1. **どこでも** - スマホ・PC・タブレット、どのデバイスからでも
2. **気楽に** - 絵文字1つ、一言だけでOK
3. **待たずに** - 2タップで記録完了、ロードを待たない
4. **誰にも見られず** - 完全プライベート、共有は任意
5. **ツールトラブルなく** - PWAでインストール不要、データはクラウド同期

---

## Duolingoから学んだこと

**122日継続できた実績**に基づく設計思想：

1. **損失回避** - ストリークが「失いたくない資産」になる
2. **最小単位が極端に小さい** - 「やらない理由」が潰される
3. **セーフティネット** - フリーズ（→ほつれ）で完全な失敗を防ぐ
4. **可視化** - 記録が目に見える形で残る

> 「最低限しかやらなくなったり、たまにフリーズ使って延命するが、122日続いてる（始めてからずっと）」

この「最低限でも続く」がポイント。日記アプリの目的は「立派な日記を書くこと」ではなく「記録を途切れさせないこと」。

## 過去の挫折：Obsidianでの記録

### 期間
2025年7月〜10月

### 継続状況
- 7/14〜8/9：約4週間、ほぼ毎日継続
- 8/9以降：急激に頻度低下（hogeのまま放置されたファイル多数）
- 10/3が最後

### データから見えた事実
- ファイル自体は作成されていた（テンプレート自動生成）
- 中身が「hoge」のまま＝**入力のハードルが問題**
- 約1ヶ月で脱落＝**新規性の報酬が切れるタイミング**

### 挫折要因
1. GitHub同期でトラブル
2. スマホでの操作が不便（ディレクトリ構造）
3. デザインが堅苦しい
4. メンタル落ちると書けない
5. 継続の意味を見失う（ゴールがない）

### 良かった点
- 文量に拘らず雑多に箇条書きで書く構成
- 文章で書きたいときだけ書く場所がある

## サービス名の由来

**ヒビオル**はアニメ映画『さよならの朝に約束の花をかざろう』に登場する、イオルフの民が織る布の名前。

縦糸が「月日」、横糸が「人のなりわい」を表し、日々の出来事や人生そのものを意味する。毎日の断片を織り重ねていくイメージから名付けた。

### 関連用語

**ほつれ** - 継続を守るためのスキップ機能

「フリーズ」は冷たい印象があるため、織物のメタファーに合わせた。
- 途切れても「ほつれ」程度、大したことない感じ
- ネガティブすぎず、でも「本来は続いてた」感がある

## 技術選定の理由

### Next.js + Supabase + Vercel

| 選定 | 理由 |
|------|------|
| Next.js | Nuxt経験あり、React系に挑戦。App Routerで最新構成 |
| Supabase | Auth・DB・Realtimeが一発で揃う。無料枠が広い |
| Vercel | Next.jsとの親和性、デプロイが楽 |
| PWA | あらゆるデバイスから入力したい。ネイティブは開発コスト高 |

### Supabase無料枠の検討

| 項目 | 無料枠 | ヒビオルでの使用見込み |
|------|--------|----------------------|
| DB | 500MB | 日記なら数年分OK |
| Storage | 1GB | 100KB/枚で約10,000枚（10年分） |
| Egress | 10GB/月 | 画像表示頻度に注意 |

**注意点**: 1週間非アクティブで自動停止
→ 本番運用時はProプラン（$25/月）に移行

## ソーシャル機能の設計思想（やらない設計）

ADHDの報酬系を余分に刺激しないため、意図的に排除した機能がある。

### 排除した機能と理由

| 機能 | 排除理由 |
|------|---------|
| **DM機能** | 返信義務感がストレスになる。記録に集中できなくなる |
| **通知数バッジ** | 数字が気になって他のことに集中できない |
| **フォロワー数の外部表示** | 数字の比較がストレスになる |
| **無限スクロールフィード** | ドーパミンループに陥りやすい |
| **いいね数の表示** | 承認欲求の刺激を避ける |
| **おすすめ・トレンド** | 過剰な情報摂取を避ける |

### 守っている原則

1. **SNS化しない** - 無限スクロール、いいね数表示、おすすめなし
2. **比較・競争を煽らない** - フォロワー数は本人にのみ表示
3. **報酬は「継続」に対してのみ** - 達成（アチーブメント）システム
4. **一方向フォロー** - 承認なしでフォロー可能、シンプルな関係性

### 実装済み機能

- **達成（アチーブメント）**: 連続記録、累計投稿数、1日の投稿数、初共有の4種類
- **共有投稿**: 特定の投稿だけをフォロワーに公開
- **お祝い機能**: 達成に対してワンタップで応援
- **ソーシャルフィード**: フォロー中ユーザーの達成・共有投稿を表示
- **プロフィール管理**: ユーザー名・表示名の設定

## 個人開発リサーチの進め方（メモ）

### Phase 1: 自分を被験者にする
- 既存ツールでプロトタイプ運用
- 「何が面倒か」「何で挫折するか」を記録

### Phase 2: 同じ課題を持つ人を探す
- X/Redditで検索（「ADHD 日記 続かない」等）
- 既存ツールへの不満を収集
- 行動ベースの情報を重視（アンケートは信用しすぎない）

### Phase 3: 競合調査
- App Store、Product Huntで検索
- レビューで「何が褒められ、何が批判されているか」
- ADHD向けアプリ：Tiimo、Routinery、Structured、Finch

### Phase 4: 最小限で公開
- MVP（機能を絞りに絞る）
- ランディングページで需要を測る

# マーケティング

## ターゲット市場

### ADHD当事者・傾向のある人

| 指標 | 数値 |
|------|------|
| 日本のADHD有病率 | 成人の約2.5〜5% |
| 日本の成人人口 | 約1億人 |
| 推定当事者数 | 250〜500万人 |
| 診断済み | 少数（未診断が多い） |

「ADHDっぽい」「続けられない」層まで含めるとターゲットは広い。

## 競合分析

### 直接競合（ADHD向け）

| アプリ | 特徴 | 弱点 |
|--------|------|------|
| Tiimo | スケジュール特化、ビジュアル | 日記機能なし |
| Finch | ゲーミフィケーション、ペット育成 | 記録というより習慣 |
| Routinery | ルーティン管理 | 自由な記録に弱い |

### 間接競合（日記・ログ系）

| アプリ | 特徴 | 弱点 |
|--------|------|------|
| Day One | 高機能日記 | 重い、継続支援弱い |
| Notion | 自由度高い | 設定が必要、ADHD向けでない |
| Twitter/X | 投稿感覚は近い | 公開前提、ノイズ多い |

### ヒビオルの差別化ポイント

- 「瞬間記録 × 継続支援」の組み合わせ
- Duolingo式ゲーミフィケーション
- ADHD当事者が設計（当事者視点）

## ストーリーテリング

**「ADHDの自分が作った、ADHDのためのアプリ」**

これが最大の武器。

- 122日Duolingo継続の実体験
- Obsidianで挫折した経験
- 当事者だからわかる「続かない理由」

開発過程をオープンに発信することで：
- 共感を得る
- フィードバックをもらう
- ローンチ前からユーザーを作る

## リスクと対策

| リスク | 対策 |
|--------|------|
| 「ADHDアプリ」というラベルへの抵抗 | 「続けられない人のため」という広い訴求も併用 |
