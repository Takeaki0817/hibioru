# ドキュメントページ構築方針

## 概要

アプリケーションの使い方、法的文書、更新履歴などを掲載するドキュメントページを作成する。

## 決定事項

### アーキテクチャ

**同ドメイン（Next.js内）で実装する**

別サーバー（VitePress等）は採用しない。

### 理由

1. **シンプルさ**: 数ページ程度ならNext.js内で十分
2. **管理コスト**: 別サーバー = 別ドメイン/SSL/デプロイ設定が不要
3. **デザイン統一**: 既存のshadcn/uiデザインシステムをそのまま活用
4. **Vercel統合**: 追加ホスティング不要

## ページ構成（予定）

```
/docs              # ドキュメントトップ
/docs/guide        # 使い方ガイド
/docs/privacy      # プライバシーポリシー
/docs/terms        # 利用規約
/docs/changelog    # 更新履歴
```

## 実装方針

### レイアウト

- サイドバー + コンテンツエリア（VitePressライクなUI）
- レスポンシブ対応（モバイルではドロワー式サイドバー）

### コンテンツ管理

- **MDX**を使用してMarkdown記法で記述可能にする
- ファイルベースでコンテンツ管理

### 機能

- サイドバーナビゲーション
- 目次（Table of Contents）
- 綺麗なタイポグラフィ（prose系スタイル）
- 将来的に検索機能も検討

## 技術スタック

- Next.js App Router
- MDX（@next/mdx または contentlayer）
- shadcn/ui（既存コンポーネント活用）
- Tailwind Typography（prose）
